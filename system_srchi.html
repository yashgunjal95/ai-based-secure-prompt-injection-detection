Fig. 1. System Architecture — AI-Based Secure Prompt Injection Detection System with Enterprise LLM Agent.
The three-layer security firewall is implemented as a LangGraph StateGraph and enforces gating before agent execution.
Input Layer

② Layer 1 — Rule-Based Detection (Weight: 0.30)
RuleBasedDetector.scan(prompt)
Rule-Based Scanner
78 compiled regex patterns · 9 attack categories

Executes in < 1 ms · No external API · Zero cost
CRITICAL match → score 1.0
HIGH → 0.85 · MEDIUM → 0.65
③ Layer 2 — ML Classification (Weight: 0.35)
MLPromptClassifier.predict(prompt)
TF-IDF + Logistic Regression Pipeline
TF-IDF Vectorizer: n-gram range (1–3), max features = 8,000, sublinear_tf = True
Logistic Regression: C = 1.0, class_weight = balanced, max_iter = 1000
Output: injection probability score 0.0 – 1.0 · Latency: 3–8 ms · Local model
④ Layer 3 — LLM Semantic Validation (Weight: 0.35)
LLMSemanticValidator.validate(prompt)
LLM Semantic Validator
Model: Groq Llama-3.3-70B Versatile · Zero-shot chain-of-thought prompting
Returns: verdict (safe / suspicious / malicious), risk_score, attack_type, reasoning
Fail-safe policy: on error → risk_score = 0.5, recommendation = sanitize
⑤ Decision Engine — Score Aggregation & Routing
Final Score = L1 × 0.30 + L2 × 0.35 + L3 × 0.35 → Apply Override Rules → Route
OR-1 Catastrophic Signal
Any single layer ≥ 0.95 → floor score at 0.95
OR-2 Critical Rule Escalation
L1 = CRITICAL severity → final score = 1.0
OR-3 Unanimous Detection
All three layers > 0.55 → routing = BLOCK
OR-4 LLM Malicious Verdict
LLM verdict = malicious → floor score at 0.65

⑦ Enterprise Agent Graph — Security Gate Node
EnterpriseGraph — security_gate
Security Gate Enforcement
Reads pipeline routing from PipelineState injected as graph state
If routing = block → blocked = True → agent node never invoked → END
If routing = allow or sanitize → run_agent node is invoked with final_prompt
